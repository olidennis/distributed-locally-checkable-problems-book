\chapter{Introduction}
The goal of this book is to provide an overview of recent results about locally checkable problems in the distributed setting. 
Locally checkable problems are problem for which, if we are given a solution, we can efficiently check its validity. 
We will mainly consider the LOCAL model of distributed computing, a synchronous message-passing model, where a graph represents a communication network, and neither the size of the messages nor the computational power of the machines is restricted.
The considered complexity measure is the number of communication rounds required to solve a given problem.
We will address the following topics:
\begin{itemize}
    \item Distributed Complexity Theory. 
    \begin{itemize}
        \item What are the possible distributed time complexities of locally checkable problems?
        \item Does randomness help? How much? Is shared randomness more powerful than private randomness? How about quantum?
        \item Can we automate our work, that is, can we automatically determine the complexity of a given problem?
    \end{itemize}
    \item Lower Bound Techniques.
    \begin{itemize}
        \item We will mainly talk about the round elimination technique, which has been used to prove many lower bounds over the years.
        \item We will discuss other techniques, such as Mark's technique, KMW, and indistinguishability arguments.
    \end{itemize}
    \item Connections with other fields.
    \begin{itemize}
        \item There are deep connections between the distributed setting and a mathematical field called \emph{descriptive combinatorics}.
        \item There are deep connetions bewtween the distributed setting and \emph{finitary factors of iid processes}, a topic coming from the area of analysis of randomized processes.
    \end{itemize}
\end{itemize}


\chapter{Preliminaries}
\section{The LOCAL model}
We model a network as a graph $G = (V,E)$, where the nodes represent machines, and the edges represent communication links. Nodes may be assigned unique IDs.
This model works as follows:
\begin{itemize}
    \item Time proceeds in synchronous steps (machines share a clock).
    \item At the beginning, each node only knows its own input and its degree. The input of a node can be, e.g., its ID, the knowledge of the size $n= |V|$ of the graph, the maximum degree $\Delta$, and/or some additional problem-specific inputs.
    \item Then, computation proceeds in rounds. At each round, each node can exchange messages with its neighbors, and update its own local state (each node has its own private memory, no memory is shared between nodes).
    \item Each node must eventually produce an output.
\end{itemize}
The complexity of an algorithm is the worst-case number of rounds that it requires to terminate. This complexity is often measured as a function of $n$ and $\Delta$.

In the LOCAL model, messages can be arbitrarily large, and computation is unbounded. There are many variants of the LOCAL model that one can consider, and they may have different power:
\begin{itemize}
    \item Is $n$ known by the nodes? Is a polynomial upper bound on $n$ known?
    \item Do nodes have IDs?
    \item Do nodes have access to randomness? How much randomness? A finite or infinite bit-string? Do we require Las-Vegas or Monte-Carlo algorithms?
    \item Messages are arbitrarily large, but are they finite? Can we e.g.\ send real numbers?
    \item Can nodes locally perform uncomputable computation?
\end{itemize}


\section{Locally Checkable Problems}
Locally Checkable Problems (LCP) are problems for which it is easy to check whether a given solution is valid. In more detail, a locally checkable problem is a problem for which there exists a constant-time distributed algorithm called \emph{verifier} satisfying that, given a solution:
\begin{itemize}
    \item If the solution is correct, then the algorithm outputs \emph{yes} on all nodes.
    \item If the solution is incorrect, then the algorithm outputs \emph{no} on at least one node.
\end{itemize}
Many interesting problems are locally checkable, for example:
\begin{itemize}
    \item Coloring problems. For example, consider the $(\Delta+1)$-coloring problem, and assume that the verifier knows $\Delta$. The verifier requires $1$ round of communication, in which it checks that the color of the current node is in $\{1,\ldots,\Delta+1\}$, and that it is different from the color of the neighbors.
    \item Maximal independent set. The verifier needs to check that if the current node is in, then all its neighbors must be out, and that if the current node is out, then at least one neighbor is in. Observe that the \emph{Maximum} independent set problem is not locally checkable.
    \item Single source shortest paths (SSSP), or at least some variants of it. Consider for example the variant in which each node outputs its own distance and a pointer to the parent. Then the verifier needs to check that the distance of the parent, plus the weight of the edge connecting the node to the parent, is equal to the distance of the node, and that no other neighbor would give a strictly smaller distance.
    \item Sinkless orientation, that is, orienting the edges so that each node does not have all edges oriented incoming.
    \item List-coloring, where each node is given a list of colors as input, and each node needs to output a color from its own list, such that the resulting coloring is proper.
\end{itemize}

\section{LCLs}
In \cite{naor-stockmeyer-1995-what-can-be-computed-locally}, a restricted class of LCP, called Locally Checkable Labelings (LCLs), was introduced. This class is restricted enough so that we can prove interesting results, but wide enough to still contain many problems of interest. In more detail, LCLs are LCPs with the following restrictions:
\begin{itemize}
    \item The number of input and output labels is constant (i.e., it does not depend on $n$).
    \item It is assumed that $\Delta \in O(1)$.
\end{itemize}
An example of a problem that is an LCP but not an LCL is SSSP, because nodes need to output distances, and distances could depend on $n$.

The nice thing about this class of problems is that each problem has a finite description:
\begin{itemize}
    \item Let $r$ be the number of rounds taken by the verifier. Clearly, the output of the verifier can only depend on things at distance at most $r$ from the node. 
    \item Since $\Delta$ and $r$ are in $O(1)$, and the number of labels is constant, there is a finite number of graphs that the verifier could see.
    \item Thus, we can list the ones that the verifier would accept. Hence, we can describe a problem by listing the accepted neighborhoods.
\end{itemize}

\section{Distributed Complexity Theory}
Being LCLs so restrictive, researchers managed to prove many interesting properties about them. Moreover, many techniques that have been initially developed to understand specific LCLs, have then been generalized to understand problems that fall outside of this class (e.g., round elimination).
Common questions are the following:
\begin{itemize}
    \item What are possible LCL complexities?
    \item Can we automatically decide the complexity of a given LCL?
    \item Does shared/private randomness help?
    \item Does quantum help?
\end{itemize}

\subsection{Paths and Cycles}
In the case of paths and cycles, LCLs have 3 possible complexities \cite{balliu-brandt-etal-2019-the-distributed-complexity-of,brandt-hirvonen-etal-2017-lcl-problems-on-grids,chang-studeny-suomela-2021-distributed-graph-problems}:
\begin{itemize}
    \item $O(1)$. Problems with this complexity often admit very easy algorithms. In the case of directed cycles with no inputs, we even know that all problems with this complexity can be solved in $0$ rounds.
    \item $\Theta(\log^\star n)$. Problems with this complexity require breaking symmetry, but do not require global coordination. Examples are $3$-coloring and MIS.
    \item $\Theta(n)$ and unsolvable problems. Problems with this complexity require global coordination. An example is $2$-coloring, where, by fixing the output of a single node, we are forcing the output on all other nodes.
\end{itemize}
Surprisingly, LCLs on paths and cycles cannot have any other possible complexity, and randomization never helps. This fact has the following interesting implication: you can be sloppy. Suppose you prove that your favourite problem can be solved in $O(\log n)$ rounds via a simple randomized algorithm. You automatically get that the problem can also be solved in $O(\log^\star n)$ by a more clever algorithm, deterministically.

Another surprising fact is that everything is decidable, even for LCLs with inputs. In other words, we can feed the description of an LCL to a computer program, and the program is going to output the correct complexity, and a description of an optimal algorithm.

\subsection{Trees}
In the case of trees, there are the same possible complexities of paths and cycles, plus some more \cite{chang-pettie-2019-a-time-hierarchy-theorem-for-the,balliu-brandt-etal-2018-almost-global-problems-in-the,chang-2020-the-complexity-landscape-of-distributed}:
\begin{itemize}
    \item $\Theta(\log n)$ deterministic. These problems can either be $\Theta(\log n)$ randomized, or $\Theta(\log \log n)$ randomized.
    \item $\Theta(n^{1/k})$ for any $k \in \mathbb{N}^{+}$. For these problems, randomness does not help.
\end{itemize}
No other complexity is possible. Moreover, it is decidable whether a problem can be solved in $O(\log n)$ or not, and in the latter case, what the exact value of $k$ is. Summarizing, some things are still decidable, and randomness can only help exponentially or not at all, and if it helps, it only helps for problems with deterministic complexity $\Theta(\log n)$. A major open question is whether it is decidable if a problem is $\Omega(\log n)$.

\subsection{General graphs}
In general graphs, things are entirely different. For deterministic complexities, the following is known \cite{chang-kopelowitz-pettie-2016-an-exponential-separation,balliu-hirvonen-etal-2018-new-classes-of-distributed,balliu-brandt-etal-2018-almost-global-problems-in-the}:
\begin{itemize}
    \item There are problems with complexity $O(1)$.
    \item No problem with complexity in $\omega(1)-o(\log \log^\star n)$ exists.
    \item The region $\Omega(\log \log^\star n)-O(\log^\star n)$ is dense: informally, for any complexity in this range, we can find a problem with a complexity that is arbitrarily near to it.
    \item No problem with complexity in $\omega(\log^\star n)-o(\log n)$ exists.
    \item The region $\Omega(\log n)$ is dense.
\end{itemize}
In the case of general graphs, undecidability results are known. Very informally, they are proved by showing that it is possible to define problems that require outputting the execution of a Turing machine, and whether the problem is constant or not depends on the running time of the machine. About randomness, a surprising connection is known: if a problem has randomized complexity $o(\log n)$, then it has randomized complexity $O(T_\mathrm{LLL})$, where $T_\mathrm{LLL}$ is the distributed time complexity of the constructive Lov\'asz Local Lemma \cite{chang-pettie-2019-a-time-hierarchy-theorem-for-the}. It is known that $T_\mathrm{LLL}$ is in $\Omega(\log \log n)$ and in $O(\poly \log \log n)$, and a big open question is determining the exact complexity of LLL. Moreover, it is known that randomness can help in different regions. For example, it is known that, for all $k \in \mathbb{N}$, there are problems with deterministic complexity $\Theta(\log^{k+1} n)$ and randomized complexity $\Theta(\log^k n \log \log n)$ \cite{balliu-brandt-etal-2020-how-much-does-randomness-help}.

\section{Easy vs Hard}
A central question about LCLs is to determine whether a problem is \emph{easy} or \emph{hard}, which we define as follows.
\begin{itemize}
    \item Easy problems are problems that can be solved in $O(\log^\star n)$. We call these problems easy because, in order to solve them, we do not need to exploit structural properties of the graph, and only some basic symmetry breaking is required. In some sense, these problems are truly local.
    \item If a problem cannot be solved in $O(\log^\star n)$, then we know it must have deterministic complexity $\Omega(\log n)$. If a problem falls in this latter case, then we say that the problem is hard. Problems of this type cannot be solved with local information: an algorithm running in $\Omega(\log n)$ rounds (for some large-enough constant hidden in the $\Omega$ notation) is guaranteed to see at least a node of degree $\le 2$ or a cycle. Algorithms of this type are not truly local; they need to see \emph{something} in order to be able to solve the problem. Moreover, problems of this type have randomized complexity $\Omega(\log \log n)$. Observe that this is the sweet spot to obtain high-probability guarantees: either the algorithm sees \emph{something} in the structure of the graph that it may be able to exploit, or it sees $\Omega(\log n)$ nodes. In the latter case, consider some randomized process that has constant success probability independently on each node. By seeing $\Omega(\log n)$ nodes, the algorithm will see a successful event with high probability, which it may be able to exploit.
\end{itemize}

\section{Black-White formalism}
An important technique used to prove lower bounds in the LOCAL model is the Round Elimination technique \cite{brandt-2019-an-automatic-speedup-theorem-for}. This technique does not directly work on LCLs, but requires the problem to be described with a specific language, called black-white formalism. It turns out that, at least on trees, this is not a restriction: we can convert any given LCL into a problem in the black-white formalism, and the two problems are equivalent, up to $O(1)$ additive rounds \cite{balliu-censor-hillel-etal-2021-locally-checkable}. On general graphs, problems in the black-white formalism are a strict subclass of LCLs, but most natural problems can anyway be expressed in the black-white formalism.

\chapter{Black-White Formalism}
The black-white formalism allows to define (some) LCLs in a concise way.
While there are many variants that we will consider later, the most basic setting is the following:
\begin{itemize}
    \item We assume that we are in a $(\Delta,\delta)$-biregular graph, where white nodes have degree $\Delta$ and black nodes have degree $\delta$. 
    \item There are no input labels.
    \item We need to output one label on each edge.
    \item We have a set (called \emph{white constraint}) of \emph{white allowed configurations}, which are multisets of size $\Delta$ that denote valid labelings of the edges incident to the white nodes.
    \item We have a set (\emph{black constraint}) of \emph{black allowed configurations}, which are multisets of size $\delta$ that denote valid labelings of the edges incident to the black nodes.
\end{itemize}
In other words, a problem $\Pi$ in the black-white formalism is a triple $(\Sigma, W,B)$, where $\Sigma$ is the (finite) set of possible labels, $W$ is a finite multiset, where each multiset has size $\Delta$, and $B$ is a multiset, where each multiset has size $\delta$.
Note that the definition of a problem $\Pi$ is just this, in particular $\Sigma$ is finite and there is no $n$ involved: we cannot define problems where the number of labels depends the size of the graph.

\section{Sinkless Orientation on Bipartite Graphs}
Let's see a simple example (illustrated in \Cref{fig:bipartite-so}). Consider the following setting: we have the promise that our graph is $3$-regular and $2$-colored. We want to define a problem that requires to orient the edges of the graph such that no node is a sink, that is, each node must have at least one edge oriented outwards.
We can define this problem in the black-white formalism as follows.
\begin{itemize}
    \item $\Sigma = \{\A,\B\}$
    \item $W = \{\{\A,\A,\A\},\{\A,\A,\B\},\{\A,\B,\B\}\}$
    \item $B = \{\{\B,\B,\B\},\{\B,\B,\A\},\{\B,\A,\A\}\}$
\end{itemize}
The intuition is the following:
\begin{itemize}
    \item Each edge gets exactly one label, $\A$ or $\B$. If an edge is labeled $\A$, then the edge is oriented from the white node to the black node, while if an edge is labeled $\B$, then the edge is oriented from the black node to the white node.
    \item The white constraint $W$ allows every configuration of $3$ labels that contains at least one $\A$, that is, at least one edge oriented away from the white node.
    \item The black constraint $B$ allows every configuration of $3$ labels that contains at least one $\B$, that is, at least one edge oriented away from the black node.
\end{itemize}
In order to describe problems in the black-white formalism more compactly, we use the following notation.
Instead of writing, e.g., $\{\A,\A,\A\}$, we simply write $\A \s \A \s \A$, or even $\A^3$. Hence, the white constraint becomes $\{\{\A^3\},\{\A^2 \s \B\},\{\A \s \B^2\}\}$. To make the notation even more compact, we write these three allowed configurations as a single one: $\A \s [\A\B]^2$. Think of it as a regular expression: we pick an $\A$, and then we can pick $\A$ or $\B$, twice. We call this type of configurations, \emph{condensed configurations}. Sometimes we even omit the square brackets, and we just write $\A \s \A\B^2$. This is the format that \emph{round eliminator} (the tool that automatically applies the round elimination technique) expects.
In order to describe a whole problem compactly, we omit $\Sigma$, we write a condensed configuration on each line, and we separate the white and the black constraint by an empty line. Hence, sinkless orientation is compactly defined as follows:
\begin{flalign*}
& \A \s \A\B^2 &\\
\\
& \B \s \A\B^2 &
\end{flalign*}

\section{Node-Edge Formalism}
The above example was in bipartite $2$-colored graphs. However, the same formalism can be used to define problems in standard graphs. The idea is to just take $\delta = 2$ (see \Cref{fig:node-edge}). Observe that, in this case, each black node is in the middle of an edge connecting two white nodes. Let's just ignore these black nodes and pretend they do not exist. We get the following setting:
\begin{itemize}
    \item We assume that we are in a $\Delta$-regular graph. 
    \item There are no input labels.
    \item We need to output one label on each \emph{half-edge} (node-edge pair). In other words, we need to output two labels for each edge, one from each side.
    \item We have a list of \emph{allowed node configurations}, which are multisets of size $\Delta$ that denote valid labelings of the edges incident to the nodes.
    \item We have a list of \emph{allowed edge configurations}, which are multisets of size $2$ that denote valid labelings of the edges.
\end{itemize}
This special case of the black-white formalism is called \emph{node-edge formalism}.

\section{Sinkless Orientation}
Let's see an example of a problem encoded in the node-edge formalism (see \Cref{fig:so}).
We can define sinkless orientation as follows:
\begin{flalign*}
& \O \s \I\O^2 &\\
\\
& \I \s \O &
\end{flalign*}
In other words:
\begin{itemize}
    \item The labels are $\I$ (oriented incoming) and $\O$ (oriented outgoing).
    \item Each node must have at least one $\O$, that is, one edge oriented outgoing. But also two or three outgoing edges are fine, so all choices over the regular expression $\O \s \I\O^2$, that is, $\O \s \I \s \I$, $\O \s \O \s \I$, $\O \s \I \s \O$, $\O \s \O \s \O$ are allowed. Note that $\O \s \O \s I$ and $\O \s \I \s \O$ are the same multiset, because the order does not matter.
    \item Edges must be properly oriented: if a node labels an edge $\I$, then the other endpoint of the edge must label the edge $\O$, and vice versa. We do not write $\O \s \I$ in the list of allowed configurations because it is the same multiset as $\I \s \O$, and the order does not matter.
\end{itemize}

\section{Coloring}
Let's see another example, $3$-coloring $3$-regular graphs (see \Cref{fig:coloring}).
We can define it in the node-edge formalism as follows.
\begin{flalign*}
& \R^3 &\\
& \G^3 &\\
& \B^3 &\\
\\
& \R \s \G\B &\\
& \G \s \B &
\end{flalign*}
In other words:
\begin{itemize}
    \item Nodes can be colored red, green, or blue.
    \item Nodes colored, e.g., red, must output $\R$ on each of the three incident half-edges.
    \item On the edges, any configuration not consisting of the same color twice is allowed, that is, $\R \s \G$, $\R \s \B$, and $\G \s \B$ are allowed, but, e.g., $\R \s \R$ is not.
\end{itemize}

\section{Maximal Independent Set}
Let's now consider a problem that is slightly harder to encode, Maximal Independent Set (MIS). In this problem, we need to select a subset of nodes that satisfy independence and maximality. While it may feel natural to try to encode it by using two labels ($\I$ and $\O$, which stand for ``in the set'' and ``out, not in the set''), this is not possible. 
In more detail, we could try to require nodes in the set to output $\I$ on each incident half-edge, and nodes not in the set to output $\O$ on each incident half-edge. Note that this would not work:
\begin{itemize}
    \item There are valid solutions in which two nodes not in the set are neighbors, and hence we would need to allow $\O \s \O$ on the edges.
    \item Hence, the node constraint contains the configuration $\O \s \ldots \s \O$, while the edge constraint contains the configuration $\O \s \O$.
    \item This implies that the obtained problem can be solved in $0$ rounds of communication, by outputting $\O$ everywhere.
    \item The obtained problem is not \emph{maximal independent set}, but just \emph{independent set}!
\end{itemize}
In other words, we need one more label to ensure maximality. We can define MIS in the node-edge formalism, on $3$-regular graphs, as follows.
\begin{flalign*}
& \M^3 &\\
& \P \s \O^2 &\\
\\
& \M \s \O\P &\\
& \O \s \O &
\end{flalign*}
That is, MIS is encoded as follows:
\begin{itemize}
    \item Nodes in the set output $\M^3$.
    \item Nodes not in the set need to \emph{prove} that they have at least one neighbor in the set, by outputting $\P$ towards them. On the other two edges, they output $\O$.
    \item On the edges, we can see that $\P$ is only compatible with $\M$ (that is, the configuration $\M \s \P$ is the only one containing $\P$), this ensures maximality. Then, nodes not in the set could have more than one neighbor in the set, and hence we allow $\M \s \O$. Moreover, nodes not in the set could be neighbors, and hence we allow $\O \s \O$.
\end{itemize}
See \Cref{fig:mis} for an example.

Let's call the usual MIS problem standard-MIS, and the version encoded in the node-edge formalism encoded-MIS.
It is important to notice that standard-MIS and encoded-MIS are not exactly the same problem:
\begin{itemize}
    \item Given a solution to encoded-MIS, we can immediately infer a solution to standard-MIS.
    \item However, given a solution to standard-MIS, it takes $1$ round of communication to obtain a solution to encoded-MIS, because nodes not in the set would not otherwise know where to output $\P$.
\end{itemize}
We need to keep this in mind when proving lower bounds: if we prove that encoded-MIS requires at least $T$ rounds to be solved, we only obtain a lower bound of $T-1$ rounds for standard-MIS. While in the case of MIS it does not matter (asymptotically), for some other problems it could matter, so we have to keep this in mind.

\section{Bipartite Maximal Matching}
Let's see another example in the black-white formalism, by considering the problem of computing a maximal matching on bipartite $2$-colored graphs (BMM).
Similarly as in the case of MIS, one may be tempted to encode BMM by using two labels, but this is not possible, we need at least three. We can encode BMM, on $(3,3)$-biregular graphs, as follows:
\begin{flalign*}
& \M \s \O^2 &\\
& \P^3 &\\
\\
& \M \s \O\P^2 &\\
& \O^3 &
\end{flalign*}
See \Cref{fig:bmm} for an example. The intuition is the following:
\begin{itemize}
    \item Edges that are part of the matching are labeled $\M$.
    \item Then, if a white node is not matched, it needs to prove that all its neighbors are matched, so it points to all of them, by outputting $\P$ on all its incident edges.
    \item There are two types of black nodes, matched and unmatched. If a node is matched then it has exactly one edge labeled $\M$, and then it can \emph{accept} pointers. So all its other edges could be $\P$ or $\O$. If a black node is not matched, then it needs to prove that all its white neighbors are matched. We do not need a new type of pointer for this, we can use $\O$, because all non-matched edges incident to white matched nodes are labeled $\O$.
\end{itemize}
Note that this is not the only possible way to encode BMM. One could use a more ``symmetric'' and intuitive version, as follows:
\begin{flalign*}
& \M \s \P\O^2 &\\
& \P^3 &\\
\\
& \M \s \P\O^2 &\\
& \O^3 &
\end{flalign*}
The intuition is the following:
\begin{itemize}
    \item Matched edges get label $\M$, then the other edges are either \emph{white pointers} $\P$ or \emph{black pointers} $\O$.
    \item If a node is matched, then it outputs exactly one $\M$, and then on the other edges everything else is allowed.
    \item If a white node is not matched, it outputs white pointers everywhere, that is, $\P^3$.
    \item If a black node is not matched, it outputs black pointers everywhere, that is, $\O^3$.
    \item Observe that if a white node is not matched, then all its edges are labeled $\P$, and all black configurations containing $\P$ also contain $\M$. Hence, all black neighbors of the white node are matched. The case of unmatched black nodes is symmetric.
\end{itemize}
It turns out that the two variants that we have seen are equivalent, and it is now time to discuss an important detail: who outputs the labels?
We will use the black-white formalism to prove upper and lower bounds via round elimination. In such a setting, it will be convenient to consider the following computational model:
\begin{itemize}
    \item White nodes output labels on their incident edges.
    \item Black nodes participate in the computation, but they do not output anything.
    \item Hence, it is the job of the white nodes to output labels that satisfy, at the same time, white and black constraints on all nodes.
\end{itemize}

Now, a solution for the first variant \emph{is also} a solution for the second variant. A solution for the second variant \emph{can be converted}, without communication, into a solution for the first variant, if we consider the computational model just discussed: matched white nodes just need to change all their $\P$ into $\O$. It is easy to see that the obtained solution is valid.

As a side note, if we enter the second variant of BMM into round eliminator, it automatically transforms the given problem into the first variant, because it sees the $\P$ in the condensed configuration $\M \s \P\O^2$ of the white constraint as useless.

\section{Ruling Sets}
Ruling sets are a family of problems parameterized by two positive integers $\alpha$ and $\beta$. An $(\alpha,\beta)$-ruling set is a set of nodes that satisfies two properties:
\begin{itemize}
    \item Nodes in the set are at distance at least $\alpha$ from each other.
    \item Each node not in the set has a node in the set at distance at most $\beta$.
\end{itemize}
Observe that the problem of computing a $(2,1)$-ruling set is the same as MIS. We will now see how to encode $(2,2)$-ruling sets on $3$-regular graphs, in the node-edge formalism.
\begin{flalign*}
& \M^3 &\\
& \P \s \O^2 &\\
& \Q \s \V^2 &\\
\\
& \M \s \P\O &\\
& \O \s \Q &\\
& \O\V^2 &
\end{flalign*}
In other words:
\begin{itemize}
    \item Nodes in the set output $\M^3$, and $\M^2$ is not in the edge constraint, to satisfy independence.
    \item Nodes not in the set need to output \emph{pointers} to prove maximality. These pointers can be $\P$ or $\Q$. However, $\P$ is only compatible with $\M$, and hence nodes outputting $\P \s \O^2$ must be at distance $1$ from at least one node in the set. Nodes at distance $2$ from nodes in the set output $\Q \s \V^2$, and $\Q$ must point to an $\O$, and hence to a node at distance $1$, ensuring that all nodes outputting $\Q \s \V^2$ are at distance $2$ from nodes in the set. 
    \item Except for needing to output pointers, there are no further constraints, so on the edges we allow any choice over $\O\V^2$.
\end{itemize}
This is not the only way to encode $(2,2)$-ruling sets. We could have for example allowed $\M \s \Q\V$ on the edges, which would allow nodes outputting $\Q \s \V^2$ to be a distance $1$ from nodes in the set.
With both encodings, it would take $2$ rounds to convert a solution for (standard) $(2,2)$-ruling sets into the version encoded in the node-edge formalism.

If we were to generalize this idea to $(2,\beta)$-ruling sets, for larger $\beta$, we would essentially do it as follows:
\begin{itemize}
    \item We require nodes in the set to output $\M^3$.
    \item For each $1\le i\le \beta$, we would introduce two new labels $\P_i$ and $\O_i$.
    \item Then, nodes at distance $i$ from the set need to output $\P_i \s \O_i \s \O_i$, and $\P_i$ would need to point to $\O_{i-1}$ (or to $\M$, if $i=1$).
\end{itemize}
However, now observe that, in order to convert a $(2,\beta)$-ruling set into a solution of this encoded version, it would take $\beta$ rounds of communication, because nodes not in the set need to figure out their distance from the set.
We need to be very careful about this when proving lower bounds: suppose we use round elimination to prove an $f(\beta)$ lower bound for the version of the problem encoded in the node-edge formalism, for some function $f$. This would not give an $f(\beta)$ lower bound for $(2,\beta)$-ruling sets, but only an $f(\beta) - \beta$ lower bound. Depending on $f$, this could matter a lot.

\section{Problems vs Family of Problems}
Observe that all the considered examples were problems defined on $3$-regular graphs. In fact, we used sentences like ``let's see how to define, in the node-edge formalism, the problem of computing an MIS on $3$-regular graphs''. Observe that we never defined ``MIS'' or ``Maximal Matching'', we always only defined them on graphs of a fixed degree.
In fact, we could define (by considering a suitable generalization of the node-edge formalism) ``MIS on graphs of degree $\le 100$'', but we \emph{cannot} define MIS in the node-edge formalism. The reason is that there are infinite values of $\Delta$, but we want our problems to have finite description (so that we can apply round elimination on them).

The workaround is the following. We don't consider MIS to be a problem, but we consider it to be a \emph{family of problems}.
In other words, we define MIS (on regular graphs) as the family of problems, containing, for each $\Delta \in \mathbb{N}^+$, the following problem:
\begin{flalign*}
& \M^\Delta &\\
& \P \s \O^{\Delta-1} &\\
\\
& \M \s \O\P &\\
& \O \s \O &
\end{flalign*}
Yes, we just replaced $\Delta$ with a $3$.
However, think about defining, e.g., $\Delta$-coloring. Then it is not as easy as making some exponents depend on $\Delta$. We would now have that the number of configurations itself depends on $\Delta$, so we would need a new language for describing our problems.
This makes it hard to apply the round elimination technique: as we will see, this technique, as is, can be applied on a given problem, but if we want to apply it on an entire family of problems at once, things get more complicated. 
However, this also shows how we can step outside the LCLs world:
\begin{itemize}
    \item According to the notation that we just introduced, MIS is definitely \emph{not} an LCL. It is an infinite family of problems, one for each maximum degree $\Delta$.
    \item We use round elimination to prove, for each problem in the family, a lower bound.
    \item We see how the asymptotics goes, and we get lower bounds as a function of $\Delta$.
\end{itemize}

\chapter{Round Elimination}
Round elimination is a technique that can be used to prove lower (and upper) bounds in the LOCAL model. Informally, round elimination is a function $\re$ that takes as input a problem $\Pi$ in the black-white formalism and outputs a new problem $\Pi'$ in the black-white formalism, with the guarantee that $\Pi'$ is exactly one round easier than $\Pi$. This statement is omitting many details that we will see later. However, lower bounds are essentially obtained by applying this function multiple times.

\section{The function $\re$.}
Let $\Pi = (\Sigma,W,B)$. We now see how the problem $\Pi' = (\Sigma',B',W') = \re(\Pi)$ is formally defined. We will later then see some intuition and some examples.
First of all, as mentioned earlier, when describing a problem in the black-white formalism, we are implicitly specifying \emph{who} is going to output the labels. That is, while the problems $(\Sigma,W,B)$ and $(\Sigma,B,W)$ have exactly the same constraints, in the former case we require white nodes to output labels, and we say that white nodes are active, while black nodes are passive, while in the latter case we require the black nodes to output, and we say that black nodes are active and white nodes are passive. When applying the round elimination function, not only we obtain a problem that is exactly $1$ round easier, but we also flip the roles of active and passive. 
Let's now see the definition of $\Pi'$. Let $\Delta$ be the degree of the white nodes, and $\delta$ be the degree of the black nodes.

\begin{itemize}
    \item $\Sigma'$ is going to be a subset of $2^\Sigma \setminus \emptyset$.
    \item We first define $B'$ as a function of $B$. 
    \begin{itemize}
        \item We first define an auxiliary constraint $\ccs$ as the set of all configurations $\{L_1, \ldots, L_\delta\}$ such that, for all $1 \le i \le \delta$, it holds that $L_i \subseteq 2^\Sigma \setminus \emptyset$, and such that, for all choices $\ell_1 \in L_1, \ldots \ell_\delta \in L_\delta$, the configuration $\{\ell_1, \ldots, \ell_\delta\}$ is in $B$.
        \item We define $B'$ as the maximal configurations in $\ccs$, that is, the subset of $\ccs$ of configurations $\{L_1,\ldots,L_\delta\}$ such that there is no other configuration $\{L'_1,\ldots,L'_\delta\}$ in $\ccs$ such that there is a permutation $\phi$ of $\{1,\ldots,\delta\}$ such that:
        \begin{itemize}
            \item For all $1 \le i \le \delta$, $L_i \subseteq L'_i$;
            \item There is at least one $i \in \{1,\ldots,\delta\}$ such that $L_i \subsetneq L'_i$.
        \end{itemize}
    \end{itemize}
    \item $\Sigma'$ is defined as the set of labels that appear at least once in the configurations in $\ccs$, that is $\Sigma' = \bigcup_{\ccc \in \ccs} \bigcup_{L \in \ccc} \{L\}$.
    \item We now define $W'$ as a function of $W$ and $\Sigma'$. We define $W'$ as the set of all configurations $\{L_1, \ldots, L_\delta\}$ such that, for all $1 \le i \le \Delta$, it holds that $L_i \in \Sigma'$, and such that there exists a choice $\ell_1 \in L_1, \ldots \ell_\Delta \in L_\Delta$ such that the configuration $\{\ell_1, \ldots, \ell_\Delta\}$ is in $W$.
\end{itemize}

\subsection{Some terminology}
We say that a configuration $L_1,\ldots,L_\delta$ \emph{satisfies the universal quantifier} if for all choices $\ell_1 \in L_1, \ldots \ell_\delta \in L_\delta$, the configuration $\{\ell_1, \ldots, \ell_\delta\}$ is in $B$. Hence we can say that $\ccs$ is the set of all configurations that satisfy the universal quantifier.
We say that a configuration is maximal if it is included in $B'$.
A configuration $L_1,\ldots,L_\delta$ is \emph{dominated} by $L'_1,\ldots,L'_\delta$ if there exists a permutation $\phi$ such that: for all $1 \le i \le \delta$, $L_i \subseteq L'_i$, and there is at least one $i$ for which the inclusion is strict. Note that $B'$ is obtained by removing dominated configurations from $\ccs$.

Similarly, we say that a configuration $L_1,\ldots,L_\delta$ \emph{satisfies the existential quantifier} if there exists a choice $\ell_1 \in L_1, \ldots \ell_\delta \in L_\delta$ such that the configuration $\{\ell_1, \ldots, \ell_\delta\}$ is in $W$.

We can now describe $\Pi'$ more compactly as follows:
\begin{itemize}
    \item $B'$ is the set of maximal configurations over the alphabet $2^{\Sigma} \setminus \emptyset$ that satisfy the universal quantifier.
    \item Let $\Sigma'$ be the set of labels appearing in $B'$.
    \item $W'$ is the set of configuration over $\Sigma'$ that satisfy the existential quantifier.
\end{itemize}



\subsection{Some intuition}
Let's unpack the definition and see one piece at a time.
It is also useful to see the proof of why $\Pi'$ is exactly one round easier than $\Pi$, to better understand why the definition is the one given.
First of all, if we defined $B' = \ccs$ (instead of taking only maximal configurations), the obtained problem would still be exactly one round easier. The only reason why we made the definition a bit more complicated is that it often makes the resulting problem ``smaller'', and hence easier to understand. So let's start by understanding $\ccs$. Think of the following:
\begin{itemize}
    \item We have an algorithm $\mathcal{A}$ that solves $\Pi$ in $T$ rounds, such that the output is produced by white nodes. We call this algorithm a \emph{white algorithm}.
    \item Recall that an algorithm in the LOCAL model is just a function that maps the view of a node (that is, its radius-$T$ neighborhood) into an output.
    \item Hence, a white algorithm does not have to be executed necessarily by a white node. If a black node $u$ gathers the view of a white node $v$, node $u$ can ask ``what would $v$ output?''
    \item We define a \emph{black algorithm} $\mathcal{A}'$ that takes $T-1$ rounds as follows:
    \begin{itemize}
        \item Each black node $u$ spends $T-1$ rounds to gather its radius-$(T-1)$ neighborhood $B_{T-1}(u)$.
        \item Observe that, if node $u$ gathered  $B_{T+1}(u)$ instead of $B_{T-1}(u)$, node $u$ could answer the following question for each white neighbor $v$: what would $v$ output on the edge $\{u,v\}$? This is doable because $B_{T+1}(u)$ contains $B_{T}(v)$, and $\mathcal{A}$, when running on $v$, solely depends on $B_{T}(v)$.
        \item We would like node $u$ to answer such a question, but node $u$ does not have enough information to answer it. However, node $u$ can answer the following question (See \Cref{fig:re-simulation}). Consider all possible ways of ``extending'' $B_{T-1}(u)$ such that $B_{T}(v)$ becomes fixed. For each such way, what is the output of $v$? By answering this question, node $u$ would obtain a set of outputs, and we can now try to infer some properties about such outputs. For now, this explains why the labels of $\Pi'$ are sets of labels of $\Pi$. 
    \end{itemize}
    \item Recall that node $u$ answered the question for each neighbor $v_1,\ldots,v_\delta$. So node $u$ now obtained $\delta$ sets $L_1, \ldots, L_\delta$, and we can ask, what do these sets satisfy? Could it be that there exists a choice $\ell_1 \in L_1, \ldots, \ell_\delta \in L_\delta$ such that $\{\ell_1,\ldots,\ell_\delta\} \notin B$? We will see that the answer is \emph{no}, and observe that $\ccs$ is defined as the constraint containing all configurations \emph{not} satisfying this property. In other words, we say, ``we don't know what these sets are going to satisfy, but they definitely do not satisfy this property, so let's take the \emph{easiest} problem not satisfying this property'' (recall that $\ccs$ contains \emph{allowed} configurations, so by putting \emph{all} configurations \emph{not} satisfying this property, we make the problem as \emph{easy} as possible). Suppose for a contradiction the answer is \emph{yes}. Then, we can take $B_{T-1}(u)$, and $\delta$ extensions of it, $Y_1, \ldots, Y_\delta$, and create a graph $G$ satisfying the following: when running $\mathcal{A}$, node $v_1$ outputs $\ell_1$, \ldots, node $v_\delta$ outputs $\ell_\delta$. We would obtain that $\mathcal{A}$ produces an invalid configuration on $u$, a contradiction. See \Cref{fig:re-contradiction}.
    \item Ignore for a moment that $\Pi'$ is defined by taking only maximal configurations from $\ccs$, and supposed we defined $\Pi'$ by using $\ccs$ directly. We obtained that $\Pi'$ can be solved in $T-1$ rounds by a black algorithm! This means that $\Pi'$ is \emph{at least one round easier} than $\Pi$.
    \item Can we also solve the \emph{real} $\Pi'$ in $T-1$ rounds, the one that does not allow non-maximal configurations? Actually, yes. Suppose node $u$, by computing $\{L_1,\ldots,L_\delta\}$, obtains a non-maximal configuration. Consider an arbitrary maximal configuration $\{L'_1,\ldots,L'_\delta\}$ satisfying that, for all $i$, $L_i \subseteq L'_i$. If $u$ outputs such a configuration, it is clearly still satisfying the black constraint. Moreover, since the white nodes need to satisfy an existential quantifier, and since we made each set larger, we obtain that the white constraint is also still satisfied.
    \item Let's now prove that $\Pi'$ is \emph{at most one round easier}. We prove this by showing that, given a black algorithm $\mathcal{A'}$ solving $\Pi'$ in $T-1$ rounds, we can create a white algorithm $\mathcal{A}$ solving $\Pi$ in $T$ rounds. The algorithm $\mathcal{A}$ simply works as follows. 
    \begin{itemize}
        \item Run $\mathcal{A'}$. This takes $T-1$ rounds.
        \item Each black node $u$ sends to each white neighbor $v$ the output that it produced for the edge $\{u,v\}$. This only takes $1$ round. 
        \item Each white node obtained sets $L_1,\ldots,L_\Delta$. By the definition of $W'$, white nodes can pick $\ell_1 \in L_1,\ldots,\ell_\Delta \in L_\Delta$ such that $\ell_1,\ldots,\ell_\Delta$ is in $W$, and output such a configuration. By the definition of $B'$, since the configurations in $B'$ satisfy the universal quantifier, all black nodes will be happy.
    \end{itemize}
\end{itemize}

\section{Some technicalities}
Recall that the core idea used for proving that $\Pi'$ is at least one round easier than $\Pi$ is the following:
\begin{itemize}
    \item Assume that, for each $i$, there is an extension $Y_i$ that, when combined it with $B_{T-1}(u)$, forces $v_i$ to output $\ell_i$.
    \item Then, we can create a graph $G$ that contains $B_{T-1}(u)$ and all $Y_i$ \emph{at the same time}. Recall \Cref{fig:re-contradiction}.
\end{itemize}
However, can we really do that? Imagine that the $Y_1$ that forces $v_1$ to output $\ell_1$ contains a node with id $123$, and that the $Y_2$ that forces $v_2$ to output $\ell_2$ contains also a node with id $123$. Then the graph $G$ that we create would not be a valid instance, because it would contain two distinct nodes with the same id.
This is the reason why round elimination does not directly apply to the LOCAL model, and some more machinery is needed.
As is, round elimination works in the so-called \emph{deterministic port-numbering model}, which is the same as deterministic LOCAL, but without IDs. This is a weak model, and we would not be satisfied if our lower bounds applied to just this model. There is another issue: suppose $T \gg \log_\Delta n$. For the graph to stay regular, in $T$ rounds of communication, we would necessarily have to see at least one cycle. This could also make $Y_1$ and $Y_1$ not independend of each other. This is why round elimination cannot give lower bounds that are asymptotically better than $\log_\Delta n$.

While how to get lower bounds better than $\log_\Delta n$ is still unknown, the other issue is fully solved. The route is the following:
\begin{itemize}
    \item We first make round elimination work in the \emph{randomized port-numbering model}, which is the same as randomized LOCAL but without IDs. In fact, one can prove that, if there is a white algorithm that solves $\Pi$ in $T$ rounds with failure probability $p$, then there is a black algorithm that solves $\Pi$ in $T-1$ rounds with failure probability $p'$ that is polynomially larger than $p$. This polynomial evolution in the failure probability is what makes randomized lower bounds obtained via round elimination asymptotically not better than $\log_\Delta \log n$.
    \item Then, we observe that randomized port-numbering and randomized LOCAl are essentially the same model: in the randomized port-numbering model we can generate unique ids with high probability. Thus, we obtain lower bounds for randomized LOCAL.
    \item Then, we use some existing techniques to convert a randomized lower bound into a lower bound for deterministic LOCAL that is even better.
    \item However, we really don't have to worry about all this! There are existing theorems that can be used fully as a black box, to convert a lower bound obtained via round elimination into lower bounds for the LOCAL model. We will discuss this in more detail later.
\end{itemize}
